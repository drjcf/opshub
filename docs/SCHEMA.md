# OpsHub — Firestore Schema v0.1
Multi-tenant AAAHC compliance & practice-management hub.
Standalone deployment = one org doc; `ORG_ID` set in app config. Same codebase.
Codename "opshub" is a placeholder.

Conventions:
- All timestamps are Firestore Timestamps, server-set where noted.
- No PHI anywhere. Incident/QI schemas use structured non-identifying fields only.
- Evidence and audit collections are append-only. Corrections = new version + supersession link. No deletes.
- Actor attribution is always `{ uid, displayNameSnapshot, at }` — snapshot the name; don't join at read time.

---

## Top level (global reference data, read-only to clients)

### standardsEditions/{editionId}
Edition of the AAAHC handbook, e.g. `aaahc-2026-v43`.
```
label:        "AAAHC Accreditation Handbook, v43 (2026)"
programType:  "ambulatory" | "asc-medicare-deemed" | "office-based-surgery"
effectiveFrom: Timestamp
status:       "current" | "superseded"
```
IMPORTANT: no verbatim AAAHC text anywhere in this tree. Citations only.

### standardsEditions/{editionId}/standards/{standardId}
Doc ID = normalized citation code, e.g. `10-I-C`.
```
code:        "10.I.C"            // display citation
chapter:     10
chapterName: "Surgical and Related Services"   // chapter names are OK; element text is not
subchapter:  "I"
element:     "C"
shortRef:    "Crash cart / emergency equipment readiness"  // your paraphrase, not AAAHC text
tags:        ["safety", "equipment"]
sortKey:     "10.09.03"          // zero-padded for ordered chapter browsing
```

## orgs/{orgId}

Org document:
```
name, legalName, facilityType: "clinic" | "asc" | "combined"
accreditation: { body: "AAAHC", editionId, programType, surveyWindow: {from,to} | null }
timezone:     "Pacific/Honolulu"
licenseKey:   string             // your licensing hook; validated by callable at boot
status:       "active" | "suspended"
createdAt
```

### orgs/{orgId}/members/{uid}
UI/read model for people. Authorization truth lives in custom claims (see AUTH below); this doc mirrors it for display and lookups.
```
displayName, email, title
roles:       ["owner"|"admin"|"clinicalDirector"|"staff"|"surveyor"]
moduleGrants: { credentialing: "rw", qi: "ro", ... }   // optional per-module overrides
active:      bool
```

### orgs/{orgId}/obligations/{obligationId}
The engine. A recurring or one-time compliance requirement.
```
title:        "Crash cart check"
description
standardRefs: [ { editionId, standardId } ]   // pinned to edition
cadence:      rrule string | null             // null = one-time
evidenceType: "checklist" | "minutes" | "log" | "document" | "drill" | "study"
checklistTemplateId: string | null
assignedRole: string                          // role, not person — people churn
assignedUid:  string | null                   // optional pin to individual
gracePeriodDays: number
escalation:   [ { afterDays, notifyRole } ]
status:       "active" | "retired"
createdBy:    Actor, createdAt
```

### orgs/{orgId}/tasks/{taskId}
Materialized instance of an obligation, generated by scheduled function.
```
obligationId, title (denormalized), standardRefs (denormalized)
dueAt, graceUntil
status:      "open" | "complete" | "missed" | "waived"
completedBy: Actor | null
evidenceId:  string | null        // set on completion
waiver:      { reason, approvedBy: Actor } | null   // waivers are audit gold — never delete
generatedAt
```
Status transitions server-side only (callable). `missed` is set by the scheduler,
never edited away — a missed task with a late evidence artifact tells the true story,
which is exactly what a QI program is supposed to surface.

### orgs/{orgId}/evidence/{evidenceId}
The atomic primitive. Append-only.
```
type:         "checklist" | "minutes" | "log" | "document" | "drill" | "study" | "attestation"
title
standardRefs: [ { editionId, standardId } ]
taskId:       string | null
payload:      map                 // typed by `type`; checklist answers, drill details, etc.
attachments:  [ { storagePath, fileName, contentType, sha256, uploadedBy: Actor } ]
status:       "draft" | "finalized" | "superseded"
finalizedBy:  Actor | null        // set by callable, acts as e-signature
finalizedAt:  Timestamp | null
supersededBy: evidenceId | null
supersedes:   evidenceId | null
createdBy:    Actor, createdAt
```
Rules enforce: editable only while `draft`; `finalized` docs immutable;
supersession only via callable, which creates the new version and links both.

### orgs/{orgId}/documents/{docId}          — Document Control (policies/procedures)
```
title, category: "policy" | "procedure" | "bylaws" | "plan" | "jobDescription"
standardRefs
currentVersionId
reviewCadence: rrule              // annual review is an auto-obligation
nextReviewDue
status: "active" | "retired"
```
#### orgs/{orgId}/documents/{docId}/versions/{versionId}
```
versionNumber, storagePath, sha256
changeSummary
approval: { approvedBy: Actor, gbMinutesEvidenceId }   // ties approval to GB minutes
effectiveAt
status: "draft" | "approved" | "superseded"
```

### orgs/{orgId}/credentialFiles/{providerId}
```
providerName, npi, category: "physician" | "crna" | "rn" | "allied"
privileges: [ { privilege, grantedAt, basis, expiresAt } ]
reappointmentDue
```
#### .../credentialFiles/{providerId}/items/{itemId}
License, DEA, board cert, PSV record, peer reference.
```
itemType, issuer, identifierMasked, verifiedBy: Actor, verifiedAt
expiresAt | null                  // expirations auto-generate obligations
evidenceId                        // the PSV proof artifact
```

### orgs/{orgId}/personnel/{staffId}
Mirrors credentialFiles for non-privileged staff: certifications (ACLS/BLS),
competencies, orientation checklist, occupational health attestations
(structured yes/no attestation only — no health details).

### orgs/{orgId}/qiStudies/{studyId}
Typed to AAAHC's 10-element format — schema forces compliant shape:
```
elements: {
  purpose, goal, performanceMeasure, dataSource,
  baseline, benchmark, findings, correctiveAction,
  remeasurement, conclusion
}                                  // each: { text, completedAt | null }
status: "planning" | "active" | "remeasuring" | "complete"
evidenceIds: []                    // supporting artifacts
cycleId: "2026-2028"              // two complete studies per cycle is the obligation
```

### orgs/{orgId}/committees/{committeeId}
GB, MEC, QI, Safety, Infection Control.
```
name, charterDocId, cadence: rrule, membership: [uid|external name]
```
#### .../committees/{committeeId}/meetings/{meetingId}
```
scheduledFor, agenda: [items]
minutesEvidenceId                  // minutes are evidence artifacts
actionItems: [ { text, assignedRole, taskId } ]   // loop back into task engine
```

### orgs/{orgId}/assets/{assetId}
```
name, category: "biomed" | "lifeSafety" | "sterilization" | "it"
location, serialMasked
pmCadence: rrule → generates obligations
lastServiceEvidenceId
```

### orgs/{orgId}/incidents/{incidentId}
Structured, de-identified by schema — free-text is bounded and labeled.
```
eventDate (date only, no time-of-day if it could identify)
category: enum (fall, medication event, equipment, transfer, grievance, ...)
severityScale, factualSummary (bounded length, no-identifier policy flag)
rcaEvidenceId | null
status: "open" | "underReview" | "closed"
committeeReviewMeetingId
```

### orgs/{orgId}/auditLog/{entryId}
Admin-SDK-written only. Clients cannot write, ever.
```
actor: Actor | "system"
action: "evidence.finalize" | "task.complete" | "doc.approve" | ...
targetPath, before: map|null, after: map|null
at: serverTimestamp
```

## AUTH (custom claims)
```
{ orgId: "ferguson-clinic",
  roles: ["owner","admin"],
  surveyorUntil: epochMillis | absent }
```
- Claims are the authorization source of truth; `members/{uid}` mirrors for UI.
- Surveyor access is time-boxed by claim: rules check `request.time < surveyorUntil`.
  Issue/revoke via callable; no standing surveyor accounts.
- Standalone deployments still set orgId claim — code never branches on tenancy.

## STORAGE
`gs://{project}.appspot.com/orgs/{orgId}/evidence/{evidenceId}/{fileName}`
Storage rules mirror Firestore org membership; sha256 recorded at upload,
verified by function — attachment integrity is part of the provenance story.

## CALLABLES (write choke points — the only paths to protected transitions)
- evidence.finalize / evidence.supersede
- task.complete / task.waive
- document.approveVersion
- surveyor.grant / surveyor.revoke
- license.validate (boot-time licensing hook)
- scheduler: materializeTasks (Cloud Scheduler → function, daily per-org tz)
